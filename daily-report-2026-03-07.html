遇到的问题

Twitter API 认证问题（CSRF Cookie 不匹配）

问题描述：xreach 读取推文时出现 CSRF cookie 不匹配错误，无法访问 Twitter/X 内容。

根因分析：Twitter Cookie 存储在 ~/.agent-reach/config.yaml，但 Cookie 值已过期或丢失。之前配置过但未保存到文件，系统重置后丢失。

解决方案：需要爸爸在 Chrome 登录 Twitter 后运行 xreach auth extract --browser chrome 提取最新 Cookie，保存到 .env 文件永久保存。

⚠️ 待处理 - 需要爸爸配合

Agent-Reach 技能"消失"问题

问题描述：之前配置好的 Agent-Reach 技能突然无法使用。

根因分析：1) 不是 OpenClaw skill（不在 skills/ 目录）；2) 没有配置在 MCP 中（不在 mcporter.json）；3) 可能是通过其他方式临时安装的，后来被清理了。

解决方案：已永久安装到 skills/agent-reach/，配置到 MCP 服务器列表，Python 依赖已安装。Cookie 待配置。

✅ 已解决 - Cookie 待配置

Brave Search API 未配置

问题描述：web_search 工具需要 Brave API key，目前无法使用。

影响：新闻搜索效率略低，需要使用浏览器搜索替代。

解决方案：使用 multi-search-engine（17 个搜索引擎，无需 API 密钥）和 tavily-search 作为替代。如需配置 Brave，需要爸爸申请 API key。

ℹ️ 可优化 - 非必需
